How this MMM works
A plain-language guide to the model's methodology, assumptions, context adjustments, and known limitations — so you can trust the numbers and know when not to.
Step 01
Adstock transformation
Raw weekly spend is a poor predictor of outcomes because advertising has carry-over effects — an impression this week still influences a purchase next month. Adstock smooths the spend series using exponential decay:

adstock[t] = spend[t] + decay × adstock[t-1]

# decay = 0.0 → no carry-over (each week independent)
# decay = 0.9 → very long memory (most past spend still counts)

# Katalon values:
# Google → 0.25 (AI Overview disruption shortens funnel)
# LinkedIn → 0.55 (Playwright competition needs longer education)
# Other → 0.45 (neutral baseline)
The choice of decay is the single most important assumption. Sensitivity analysis (see the Python file) sweeps decay 0.1→0.85 to check whether ROI outputs are stable — stable ROI across the sweep means the model is robust.
Step 02
Correlation-based attribution
After adstock transformation, the model computes Pearson correlation between each channel's adstocked spend and the chosen outcome (MQLs / Pipeline / ARR):

corr = cov(adstock_spend, outcome) / (σ_spend × σ_outcome)

# Only positive correlations are kept — a channel
# with negative correlation is treated as 0 weight.
# Correlations are then normalised to sum to 100%
# of the paid attribution share (78%).
Each channel's attribution share = its normalised correlation × paid attribution share (78%). The remaining 22% is assigned to organic baseline.
Step 03
Context adjustments
Raw correlations are adjusted by research-backed efficiency multipliers specific to Katalon's market in 2025:

Google −15% — AI Overviews now appear on 57% of SERPs; organic CTR dropped 58% YoY; 73% of B2B sites lost traffic. Katalon's top-of-funnel test automation queries ("best automation tool", "Selenium alternative") are precisely the type LLMs intercept.

LinkedIn +8% — Playwright surged to 45% QA team adoption. Decision-makers need explicit brand education on Katalon's AI features (TrueTest, StudioAssist) to justify the paid cost over free alternatives.

Organic raised to 22% — Gartner Visionary 2025 positioning + G2/review-site presence + LLM citations (which convert 4.4× better than organic search) drive untracked dark-funnel pipeline.
Step 04
ROI & budget optimisation
ROI for each channel = attributed outcome ÷ total spend. For monetary outcomes (pipeline, ARR) this is a direct multiplier. For MQLs it is MQLs per dollar.

roi_ch = (total_outcome × share_ch) / total_spend_ch

# ROI values are RELATIVE, not absolute.
# Use them for: channel ranking, budget split decisions.
# Do NOT use them for: precise revenue accounting.
The budget optimizer reallocates a given weekly budget proportionally to each channel's adjusted ROI — channels with higher ROI receive a larger slice.

Known limitations
What this model cannot do
No causal inference — correlation ≠ causation. A channel correlated with pipeline could be following it rather than driving it (e.g., LinkedIn spend goes up in busy quarters because the team has budget, not because it generates pipeline).

No interaction effects — the model treats channels independently. In reality, LinkedIn brand exposure likely amplifies Google conversion rates (a prospect sees a LinkedIn ad then Googles Katalon). This synergy is unmodelled.

No saturation curves — the model assumes linear returns to spend. In practice, doubling Google spend will not double MQLs; diminishing returns set in. A full Bayesian MMM with Hill transformation would capture this.
How to validate the model
R² and MAPE — the Python script computes in-sample fit diagnostics. Target R² > 0.7, MAPE < 20%. Run: python katalon_mmm.py

Adstock sensitivity sweep — run --sensitivity flag to check ROI stability across decay values. Stable ROI = robust model.

Geo/time holdout test — pause LinkedIn for 4–6 weeks and measure whether MQL drop matches the model's predicted attribution share. This is the only ground-truth validation.

Out-of-time validation — train on 2024-W01→2025-W26, test on 2025-W27 onwards. If test MAPE > train MAPE + 10pp, the model is overfitting.

How to run a channel pause test
1. Pick one channel to pause Start with LinkedIn (lower disruption risk than Google). Pause all LinkedIn spend for 4–6 consecutive weeks.

2. Measure the gap vs model prediction The model predicts LinkedIn drives — of MQL outcomes. During the pause, if MQLs drop by roughly that amount, the model is validated. A much smaller drop means LinkedIn's attribution is overstated.

3. Calculate the accuracy ratio Accuracy = actual MQL drop ÷ model-predicted drop. Between 0.8–1.2 = good. Outside that range = recalibrate the decay and efficiency parameters.

4. Resume and confirm recovery Re-enable spend and check whether MQLs recover within the model's predicted lag window (~4–6 weeks for LinkedIn).
